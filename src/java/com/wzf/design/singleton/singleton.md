饿汉式单例  

    简单实现，线程不安全；
    加synchronized实现,效率变低；
    双重检查实现，效率变高，但依旧有锁；且可以被破坏
    静态内部类实现，使用时加载；

破坏单例
```
1，通过反射破坏单例；解决：在私有的构造方法中做判断
2，通过序列化破坏单例；解决：在类中增加readResolve()方法
```
